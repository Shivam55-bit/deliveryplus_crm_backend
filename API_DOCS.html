Base URL http://localhost:5000/api

🔐 Authentication

Sabhi protected routes pe Authorization header bhejni hogi:

Authorization: Bearer <access_token>

Token login ya register se milega. Token expire hone pe /auth/refresh-token use karo.

👤 Roles

admin — Full access (job create, assign, reports)
driver — Only driver workflow APIs
customer — Limited access

Job Status Flow
Job lifecycle — ek direction mein hi move karta hai
pending → assigned → in_transit → started ⇌ paused → completed
cancelled — Admin kisi bhi stage pe cancel kar sakta hai
Rate Limits
⚡ /api/auth/* — Max 10 requests per 15 minutes (brute force protection)
⚡ All other /api/* — Max 200 requests per 15 minutes per IP 
{
  "success": false,
  "message": "Too many login attempts, please try again after 15 minutes."
}

Driver Workflow
Driver mobile app ke liye — Step by step job flow
GET /jobs/driver/my-jobs Mujhe assign saare jobs Driver

Query Params

ParamExample
statusassigned, started, completed
page1
GET /jobs/driver/today Aaj ke assigned jobs Driver

Aaj ki date ke jobs (scheduled) return karta hai. No params needed.

POST /jobs/:id/accept Step 1: Job accept karo Driver

No body needed. Status: assigned rehta hai, timeline updated.

POST /jobs/:id/start-transit Step 2: Pickup pe niklo Driver

Status: assigned → in_transit

POST /jobs/:id/arrived Step 3: Location pe pohonch gaya Driver

Status change nahi hota — sirf timeline updated. Customer ko sign karne ke liye prepare karo.

POST /jobs/:id/start Step 4: Job start + Timer start + Customer signature Driver

Request Body

{
  "startSignature": "data:image/png;base64,iVBOR...",  // required — customer ka start signature
  "termsAccepted": true                                 // required — customer ne T&C accept kiya
}

Side Effects

✅ Status → started
✅ timerStarted = current timestamp
✅ Signature save hoti hai

POST /jobs/:id/pause Timer pause karo (break) Driver

Status → paused. Pause time record hota hai billing se exclude karne ke liye.

POST /jobs/:id/resume Timer resume karo Driver

Status → started. Resume timestamp record hota hai.

POST /jobs/:id/complete Step 5: Job complete + Timer stop + End signature Driver

Request Body

{
  "endSignature": "data:image/png;base64,iVBOR...",  // required — customer ka end signature
  "completionNotes": "All items delivered safely",    // optional
  "damageReport": "Minor scratch on table",           // optional
  "extraCharges": 50,                                 // optional — extra charges add karna
  "extraChargeNotes": "Stairs surcharge",
  "fuelCharges": 20,
  "tollCharges": 15,
  "discount": 10
}

Response 200

{
  "success": true,
  "message": "Job completed.",
  "data": {
    "job": {
      "status": "completed",
      "timerEnded": "2026-04-22T13:00:00Z",
      "totalWorkedMinutes": 234,
      "billableHours": 3.9,
      "billing": {
        "laborCost": 468,
        "extraCharges": 50,
        "fuelCharges": 20,
        "tollCharges": 15,
        "discount": 10,
        "gst": 54.3,
        "totalAmount": 597.3
      }
    }
  }
}

Auto Calculated

✅ Timer stops — pause intervals exclude kiye jaate hain
✅ billableHours = (total minutes - paused minutes) / 60
✅ laborCost = billableHours × hourlyRate (ya fixedQuote)
✅ GST = 10% of subtotal
✅ Driver availability → available

Error Responses
Standard error format

  "success": false,
  "message": "Error description",
  "errors": []    // validation errors hone pe
}

// HTTP Status Codes:
// 400 — Bad Request (validation fail)
// 401 — Unauthorized (token missing/invalid)
// 403 — Forbidden (role access denied)
// 404 — Not Found
// 409 — Conflict (duplicate entry)
// 429 — Too Many Requests (rate limit)
// 500 — Internal Server Error
